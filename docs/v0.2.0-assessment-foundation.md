# Kenopsia Core v0.2.0 - Assessment Foundation

v0.2.0 introduces the first real assessment layer for Kenopsia Core. The goal is not to imitate a vulnerability scanner. The goal is to turn collected Linux host facts into explainable findings that can be scored, reported, and expanded over time.

## What v0.2.0 adds

- A reusable assessment engine under `kenopsia/assessment/`.
- Storage usage findings for filesystems above 80% and 90% utilization.
- Conservative SSH posture findings.
- Detection for legacy remote access services such as Telnet, rsh, and rexec.
- A local storage collector.
- A local security collector.
- Markdown rendering for assessment findings.
- A single-run helper script that writes both JSON and Markdown output.

## Running the assessment

From the repository root:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -e .
python scripts/run_v020_assessment.py --out output
```

Expected output:

```text
output/kenopsia-assessment-<hostname>.json
output/kenopsia-assessment-<hostname>.md
```

## Assessment scoring

The score starts at 100 and subtracts points based on finding severity:

| Severity | Penalty |
|---|---:|
| critical | 20 |
| high | 12 |
| medium | 7 |
| low | 3 |
| info | 0 |

Grades are assigned as:

| Score | Grade |
|---:|---|
| 90-100 | A |
| 80-89 | B |
| 70-79 | C |
| 60-69 | D |
| 0-59 | F |

## Design direction

This release establishes the assessment shape that future collector work can plug into:

```text
collector data -> assessment engine -> scored findings -> JSON/Markdown/HTML output
```

Future v0.2.x releases should continue improving collector quality and finding accuracy before the dashboard layer gets heavier.
